# v0.0.1 — initial public release

> Publish via GitHub **Releases → Draft a new release → tag `v0.0.1` on `main`**. The
> release pipeline (multi-arch image, SBOM, CVE gate, cosign signing, and the two
> release-only conformance suites) triggers on release **publication**, not on tag push.

First public release of **fhir-map** — a production-grade FHIR terminology and
transformation component: a deterministic `$transform` / `$translate` service plus a
versioned map store, built to sit *inside* a data pipeline.

## Capabilities
- **Terminology translation** — ConceptMap `$translate` (single + batch), instance- and
  type-level, forward and reverse.
- **Structural transformation** — StructureMap `$transform` via FHIR Mapping Language
  (FML), with a bounded execution model (timeout, recursion and body-size limits) and an
  opt-in strict / fail-loud mode (`SERVER_TRANSFORM_STRICT=true`) that surfaces the
  unambiguous silent-data-loss cases as 422 OperationOutcomes.
- **Dual-version API** — parallel R4 and R5 wire formats from the same server, data, and
  maps.
- **Versioned map store** — CRUD plus `_history` / `vread` and soft-delete for ConceptMap,
  StructureMap, and StructureDefinition.
- **HL7v2 / HPRIM parsing** — delimiter discovery from the message header, escape-sequence
  unescaping, field repetition, and explicit-null recognition.
- **Observability** — Prometheus metrics at `GET /metrics` (PHI-safe HTTP RED for every
  route plus `$transform` domain metrics).

## Security & supply chain
- Built with the pinned Go toolchain; `govulncheck` clean at release.
- All GitHub Actions pinned by commit SHA and Docker base images by digest.
- CodeQL (SAST), gosec, gitleaks, and OpenSSF Scorecard run in CI; Dependabot keeps
  dependencies current.
- Release artifacts: multi-arch image on `ghcr.io/goodeworkers/fhir-map`, CVE-scanned
  before push, with an SBOM and a keyless cosign signature.
