# v0.0.2 — security + production-hardening release

> Draft release notes. Publish via GitHub **Releases → Draft a new release → tag `v0.0.2`
> on `main`** — the release pipeline (multi-arch image, SBOM, CVE gate, cosign signing,
> and the two release-only conformance suites) triggers on release **publication**, not on
> tag push.

**Why this release matters:** the previous artifact (v0.0.1, 2026-05-27) was built with
Go 1.25.10 and is affected by two Go standard-library CVEs fixed since
([GO-2026-5039](https://pkg.go.dev/vuln/GO-2026-5039) net/textproto,
[GO-2026-5037](https://pkg.go.dev/vuln/GO-2026-5037) crypto/x509). v0.0.2 is built with
Go 1.25.11 and supersedes it. Deployments should upgrade.

## Security
- **Go toolchain 1.25.11** — fixes GO-2026-5039 (net/textproto) and GO-2026-5037
  (crypto/x509); `govulncheck` clean.
- `$transform` execution is **bounded** (timeout, recursion and body limits; see README).

## Production hardening (FHIR transform component)
- **Strict / fail-loud transform mode** (opt-in, `SERVER_TRANSFORM_STRICT=true`; default
  remains lenient and byte-identical): the two unambiguous silent-data-loss cases — a
  typed-transform **coercion failure** and a **resolved-ConceptMap-with-no-mapping**
  translate drop — now fail loud as 422 OperationOutcome (codes `value` / `not-found`)
  with PHI-conservative diagnostics.
- **Prometheus metrics** at `GET /metrics`: generic HTTP RED for every route
  (`fhirmap_http_requests_total`, `fhirmap_http_request_duration_seconds`, route-template
  labels, PHI-safe) plus `$transform` domain metrics (`fhirmap_transform_total{result,code}`,
  `fhirmap_transform_duration_seconds`).

## FHIR Mapping Language
- **Explicit `evaluate(context, expression)`** — both the expression form
  `evaluate(v, $this + 2)` and the quoted-FHIRPath form `evaluate(i, 'name')`.
- **Decimal + unary `±` numeric literals**, comparison operators, `$`-variables and
  `$index` in FML/FHIRPath.
- **Target list-mode execution** (`first`/`last`/`single`/`share`/`collate`) and FML
  lowering of source `check`, source list modes, source element `type`, and source
  `default`.
- **Inline `(expr)` evaluate shorthand** executes against the rule's variable scope.
- **Date masks**: Java-pattern tokenizer — `a` (AM/PM), `Z` (UTC `Z`/numeric offset),
  `hh`/`h` (12-hour) and unpadded `M/d/m/s` now supported; unknown tokens fail loud.
  Fixes a silent drop where offset-bearing datetimes failed to parse under
  `yyyy…Z`-style masks.

## HL7v2 / HPRIM parser maturity
- Delimiter **discovery from the message header** (no hardcoded `^~\&`), HPRIM
  delimiter-order divergence locked (`~` subfield / `^` repeater).
- **Escape-sequence unescaping** at leaf access (`\F\`, `\X..\`, `\.br\`, …).
- **Field repetition (`~`)** modeled as a list; explicit-null (`""`) recognized as a
  distinct third field state.
- HAPI mapping-engine parity work: segment handling, StructureMap import composition,
  HL7v2 component/index navigation, ER7 output serialization, `value[x]`
  shape-inference.

## CI / supply chain
- OpenSSF Scorecard gated on repo visibility (no more guaranteed-red runs while private).
- Bruno CLI pinned; conformance suites (HL7 Validator, real `tx.fhir.org`) run on release.
